## Image

```
ghcr.io/eugenioenko/autentico:latest
```

The image is a minimal Linux container with just the `autentico` binary. All configuration is via environment variables.

## Run

### Using an env file

The simplest approach — generate your secrets once with `autentico init`, save them to a `.env` file, and pass it to Docker:

```bash
docker run \
  --name autentico \
  -p 9999:9999 \
  -v autentico-data:/data \
  --env-file .env \
  ghcr.io/eugenioenko/autentico:latest
```

The `.env` file must be in the current working directory (or provide an absolute path). Add it to `.gitignore` to avoid committing secrets.

### Using inline environment variables

```bash
docker run \
  --name autentico \
  -p 9999:9999 \
  -v autentico-data:/data \
  -e AUTENTICO_APP_URL=http://localhost:9999 \
  -e AUTENTICO_LISTEN_PORT=9999 \
  -e AUTENTICO_DB_FILE_PATH=/data/autentico.db \
  -e AUTENTICO_PRIVATE_KEY=base64-encoded-pem-string \
  -e AUTENTICO_CSRF_SECRET_KEY=changeme-at-least-32-chars \
  -e AUTENTICO_ACCESS_TOKEN_SECRET=changeme-at-least-32-chars \
  -e AUTENTICO_REFRESH_TOKEN_SECRET=changeme-at-least-32-chars \
  ghcr.io/eugenioenko/autentico:latest
```

`AUTENTICO_LISTEN_PORT` is required here because `AUTENTICO_APP_URL` uses HTTPS (port 443) but the container listens on `9999`. Without it, the server would try to bind to port 443. Mount the `/data` volume to persist the SQLite database across container restarts. The RSA private key is passed via the `AUTENTICO_PRIVATE_KEY` environment variable — generate it once with `autentico init` and store it in your secrets manager.

## Environment variables

All configuration is done via [Bootstrap Settings](/configuration/bootstrap/). The most important ones:

| Variable | Required | Notes |
|---|---|---|
| `AUTENTICO_APP_URL` | Yes | Public-facing URL, used in OIDC discovery and redirects |
| `AUTENTICO_DB_FILE_PATH` | Yes | Path to SQLite database file inside the container |
| `AUTENTICO_PRIVATE_KEY` | No | Base64-encoded RSA private key PEM; if unset, an ephemeral key is used and tokens are invalidated on restart |
| `AUTENTICO_CSRF_SECRET_KEY` | Yes | At least 32 random characters |
| `AUTENTICO_ACCESS_TOKEN_SECRET` | Yes | At least 32 random characters |
| `AUTENTICO_REFRESH_TOKEN_SECRET` | Yes | At least 32 random characters |
| `AUTENTICO_LISTEN_PORT` | No | Port the server binds to; defaults to the port in `AUTENTICO_APP_URL` |

## Health check

Autentico exposes `/.well-known/openid-configuration` which returns 200 and can be used as a health check:

```dockerfile
HEALTHCHECK --interval=30s --timeout=5s \
  CMD wget -qO- http://localhost:9999/.well-known/openid-configuration || exit 1
```

## Logs

Autentico logs structured HTTP access logs to stdout. Collect them with your Docker logging driver or forward to a log aggregator.
