## What is Autentico?

Autentico is a self-hosted OpenID Connect Identity Provider built with Go. It handles the full authentication lifecycle — login, MFA, passkeys, session management, token issuance, and administration — in a single binary backed by SQLite.

A typical self-hosted IdP involves a database server, a cache layer, multiple processes, and everything that comes with keeping them running. Autentico removes that stack entirely. There is no external database to provision, no connection pool to tune, no sidecars — just one binary and one file.

The simplicity is operational, not protocol-level. Autentico implements OAuth2 and OpenID Connect correctly: Authorization Code + PKCE, token introspection and revocation, OIDC Discovery, RS256-signed JWTs, WebAuthn/passkeys, TOTP, and email OTP. Relying parties configure themselves automatically via the standard discovery document.

## Get running in 60 seconds

```bash
./autentico init --url http://localhost:9999
./autentico start
```

Open `http://localhost:9999/onboard`, set your admin username and password — you're in. No Docker, no database to provision, no config files to edit.

## Features

<CardGrid>
  <Card title="OIDC & OAuth2" icon="document">
    Authorization Code Flow with PKCE, Refresh Tokens, ROPC, Token Introspection
    (RFC 7662), Token Revocation (RFC 7009), and a full OIDC Discovery document.
  </Card>
  <Card title="Passkeys & MFA" icon="approve-check-circle">
    WebAuthn/FIDO2 passkey authentication, TOTP with in-browser QR enrollment,
    Email OTP, and trusted device recognition to reduce MFA friction on known
    devices.
  </Card>
  <Card title="Admin UI" icon="laptop">
    A built-in React dashboard for managing users, clients, and sessions.
    Runtime settings update without restarting the server.
  </Card>
  <Card title="Zero Dependencies" icon="rocket">
    Single Go binary with embedded SQLite. No Postgres, no Redis, no sidecars.
    Deploy anywhere a binary can run.
  </Card>
  <Card title="Three-Layer Config" icon="setting">
    Immutable bootstrap settings from `.env`, runtime settings in the database
    (hot-reloadable), and per-client overrides for token TTLs, auth modes, and
    more.
  </Card>
  <Card title="Production Ready" icon="shield">
    RS256 JWT signing, CSRF protection, bcrypt password and secret hashing,
    account lockout, configurable session idle timeouts, and a background
    cleanup service.
  </Card>
